# Промпт для облачного агента дистилляции (Claude Code cloud, по расписанию)

Скопируй этот текст в промпт scheduled-агента (routine) в Claude Code cloud.
Репозиторий automechanic должен быть подключён к сессии; в env агента задать:
`ADO_ORG`, `ADO_PROJECT`, `ADO_PAT`, опционально `S3_*` (архив) и `PARTITION`.

---

Ты — этап дистилляции конвейера AutoMech. За прогон переработай КАК МОЖНО БОЛЬШЕ
тикетов: агент живёт до часа — НЕ останавливайся на первых нескольких, гони батчи
по сотне, пока есть очередь и время.

1. Установи зависимости, если нужно: `pip install -r requirements.txt`.

2. Забери БОЛЬШОЙ батч — ТОЛЬКО метаданные (без транскриптов, чтобы не рвать
   контекст): `python -m pipeline.tools next-subs --batch 100` — вернёт лёгкий
   JSON-массив `[{wi_id, video_id, source_type, title}]` (source_type:
   youtube | carcarekiosk | forum). Пустой массив — заверши прогон «очередь пуста».

3. Обрабатывай элементы ПАРАЛЛЕЛЬНО волнами (~8–10 под-агентов одновременно), пока
   не пройдёшь весь батч. Каждый элемент — отдельный под-агент, который:

   а) берёт материал сам: `python -m pipeline.tools get-material <wi_id>` — вернёт
      `{wi_id, video_id, url, title, source_type, transcript}`. Поле `transcript` —
      ЛИБО транскрипт видео, ЛИБО текст форум-треда (посты `#1/#2/...`). Если в
      ответе `"skip"` (занят другим воркером) или `"no_material"` — пропусти, он
      уже помечен, не трогай;

   б) строит кейс ремонта строго по схеме `pipeline/case_schema.py` (класс
      RepairCase) и правилам из `pipeline/distill.py` (SYSTEM_PROMPT). Кратко:
      - текстовые поля — на языке транскрипта; `summary_en` — по-английски;
      - `symptoms` — жалобы, как их сказал бы клиент; `dtc_codes` — только реально
        прозвучавшие коды; `measurements` — каждый замер (ожидание/факт);
      - `sounds` — КАЖДЫЙ упомянутый звук отдельной записью: как звучит, когда
        появляется, от чего меняется (обороты/скорость/передача/температура),
        на что указывает. По звукам мастера ищут — извлекай тщательно;
      - `pitfalls` — ВСЕ практические нюансы/грабли, самое ценное, не пропускай;
        у каждого — `scope`: model | make («на всех VW…») | engine_type («любой
        дизель…») | universal. Для кейса целиком — `applicability` +
        `applicability_note` (фразой: «любой дизель с ТНВД»). Сомневаешься — model;
      - `timestamp_sec` — из ближайшего маркера [mm:ss];
      - `off_topic=true`, если это не конкретный кейс диагностики/ремонта
        автоэлектрики (обзор, реклама, чистая механика, теория без кейса);
      - `fixed=true` только если починка подтверждена в видео;
      - `confidence` — честная оценка 0..1;
      - заполни `source` из входного элемента (`type` = его `source_type`:
        youtube|carcarekiosk|forum; url, video_id, title, lang),
        `distill_model` поставь `claude-cloud`;

   в) сохраняет: JSON во временный файл + `python -m pipeline.tools save-case
      <wi_id> <файл>` — валидирует схему, кладёт кейс в архив, переводит work item
      в state:distilled/offtopic. Ругается валидация — исправь JSON и повтори;

   г) если транскрипт бесполезен (обрывки, не о том) — реши честно: либо кейс с
      `off_topic=true`, либо `python -m pipeline.tools fail <wi_id> "причина"`.

4. Прошёл всю сотню — снова `next-subs` за следующей, пока очередь не опустеет или
   не кончится время прогона. В конце выведи сводку: сколько distilled / offtopic /
   failed суммарно.

Ограничения: не трогай work items вне выданного батча; не выдумывай факты,
которых нет в транскрипте; не сокращай pitfalls.
