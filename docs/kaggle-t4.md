# Kaggle T4: что туда выносить, а что нет

Короткий ответ на вопрос «не лучше ли поднять Qwen на Kaggle T4?»

## Сначала — про путаницу «эмбеддинг вынимает смысл»

Эмбеддинг смысл **не вынимает**. В конвейере две разные ИИ-задачи:

1. **Дистилляция** (LLM, «сильная модель»): читает титры, ПОНИМАЕТ и пишет кейс —
   симптомы, первопричину, шаги, нюансы. Это творческая задача, качество зависит
   от ума модели. Проверено: minimax-m2.7 на NIM справляется отлично.
2. **Эмбеддинг** (bge-m3): берёт УЖЕ готовый текст кейса и кодирует его в вектор
   из 1024 чисел, чтобы потом искать похожие. Ничего не понимает и не пишет —
   это «хэш по смыслу». Модель крошечная (~0.6B) рядом с LLM.

Поэтому «поднять Qwen вместо эмбеддингов» — не замена: это разные ступени.

## Что даст Kaggle T4 x2 (30 GPU-часов/нед, сессии ~9-12 ч)

| Задача | Стоит ли на T4 | Почему |
|---|---|---|
| Эмбеддинги bge-m3 | ✅ **да, лучшее применение** | модель влезает целиком, тысячи текстов/час, ноль лимитов; API-квоты (CF 10k/день) перестают быть узким местом при индексации бэклога |
| Whisper large-v3 (видео без титров) | ✅ да | faster-whisper на T4 ~10-15x реалтайма; закрывает ~10-20% видео, где титров нет |
| Reranker (cross-encoder для поиска) | ✅ позже, для качества выдачи | |
| **Дистилляция Qwen'ом** | ⚠️ только как аварийный фолбэк | на T4x2 (2×16GB) влезает Qwen 14B, максимум 32B в AWQ-кванте. Это ЗАМЕТНО глупее minimax-m2.7 (230B-класс MoE) и nemotron-super, которые бесплатны через API. Плюс 34-мин транскрипт = ~25k токенов входа — на T4 это минуты на видео; 30 GPU-ч/нед ≈ 300-600 видео. Каскад бесплатных API даёт больше и умнее |

Вывод: **T4 = фабрика эмбеддингов и транскрипции, API-каскад = мозги.**
Если когда-то бесплатные API закончатся все разом — Qwen-32B-AWQ на T4 как
запасной дистиллятор, код менять не надо (тот же OpenAI-совместимый vLLM,
просто другой DISTILL_BASE_URL).

## Как поднять (готовый код: kaggle/embedding_server.py)

1. Kaggle → New Notebook → Settings: Accelerator **GPU T4 x2**, Internet **On**.
2. Ячейка 1: `!pip install -q fastapi uvicorn sentence-transformers pyngrok`
3. Ячейка 2: содержимое `kaggle/embedding_server.py`.
4. Add-ons → Secrets: `NGROK_TOKEN`, `EMBED_SERVER_KEY`, опц. `GIST_TOKEN`+`GIST_ID`.
5. Run → в логе появится `EMBED_REMOTE_URL=https://xxxx.ngrok-free.app`
   (и запишется в Gist — паттерн из твоего SmartScraper, конвейер может сам
   подхватывать свежий URL после перезапуска ноутбука).
6. В CircleCI-контекст: `EMBED_REMOTE_URL`, `EMBED_REMOTE_KEY` —
   `pipeline/embed.py` ходит на него первым, при недоступности падает на
   Cloudflare → NIM → локальную модель.

Держать 24/7 не нужно: поднимаешь на время больших батчей индексации.
Дежурный ручеёк (десятки кейсов/день) спокойно едет на Cloudflare free.

## Что на Kaggle НЕ выносим

- Титры (это не GPU-задача, и IP Kaggle — датацентровый, YouTube его режет
  так же, как CI; цепочка invidious/supadata живёт в CircleCI).
- Оркестрацию/состояния — это ADO.
- Векторную базу — Qdrant Cloud / Oracle 23ai (Kaggle-сессия смертна, БД там
  делать нельзя).
