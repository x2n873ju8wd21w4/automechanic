# Форумный краулер: зоны, вежливость, тайм-бокс, тикеты

Обходит все треды форумов, посты → R2, каждый тред → тикет ADO (child под
Epic форума, сразу `state:subs` — посты и есть «транскрипт», отдельного этапа
титров форуму не нужно). Дальше дистилляция берёт их наравне с видео.

## Зоны (чтобы не забанили IP)

`pipeline/forum_sites.py` → у каждого сайта поле `zone`. Прогон обходит сайты
ТОЛЬКО своей зоны. Раскладка:

| Зона | Сайты | Кто гоняет |
|---|---|---|
| **a** | carmasters.org (IPS, RU), drive2.ru (custom, RU) | аккаунт A, чётные часы |
| **b** | vwvortex, bimmerforums, bobistheoilguy (EN) | аккаунт B, нечётные часы |
| **c** | opinautos.com (ES) | по мере надобности |

Разные зоны на разных аккаунтах в разное время ⇒ один IP не молотит один
форум. Внутри прогона:
- **пауза на хост** (`per_host_delay`, 6с) — не чаще раза в N секунд к хосту;
- **чередование хостов** — берётся тред того хоста, что дольше всех «отдыхал»,
  поэтому carmasters и drive2 идут вперемешку, а не подряд.

## Тайм-бокс и возобновление

`--minutes 18` — прогон живёт не дольше и выходит, сохранив **фронтир** (очередь
URL) и множество «увидено» в R2 (`crawl/{zone}.json`). Следующий прогон
продолжает с места остановки. Укладываемся в бюджет CI-джоба (<20 мин),
`no_output_timeout: 22m` в конфиге — страховка.

Оценка: пауза 6с/хост, 2 хоста в зоне A вперемешку ⇒ ~1 запрос/3с ⇒ ~360
запросов за 18 мин. Тред = 1 запрос (+ пагинация листингов). То есть ~300
тредов за прогон; весь раздел BMW carmasters (≈460 тредов на 14 страницах)
разбирается за 2 прогона. `per_host_delay` можно поднять, если хост нервный.

## Запуск

```bash
# локально/дебаг: показать, что найдёт, без ADO
python -m pipeline.crawler --zone a --dry-run --max-threads 5

# боевой прогон (нужны ADO_* и S3_* в env)
python -m pipeline.crawler --zone a --minutes 18 --create-workitems

# сброс очереди зоны (начать обход заново)
python -m pipeline.crawler --zone a --reset
```

CI: воркфлоу `crawl` в `.circleci/config.yml`, параметры триггера
`flow=crawl`, `crawl-zone=a|b|c`. На аккаунте A scheduled trigger в чётные
часы с `crawl-zone=a`, на аккаунте B — нечётные с `crawl-zone=b`.

## Добавить форум / раздел

В `pipeline/forum_sites.py` → `SITES`: движок, `zone`, `seeds` (URL разделов),
`thread_re`/`listing_re` (уже есть для IPS/XenForo/vBulletin), `next_page`.
Сиды разделов можно вытащить из твоих HAR. Для сайтов, где список грузится
JS/через API (drive2, opinautos), ставь `mode="seed"` и перечисляй URL тредов
в `seeds` — полный обход раздела там делается через их API (отдельный клиент).

## Что краулер НЕ делает (по замыслу)

- Не качает картинки/видео/вложения — только текст постов (см. контент-политику
  в README §5.5). Ссылки на источник сохраняются, копии контента — нет.
- Не ходит за логин без HAR-сессии (`--har session.har`, если раздел закрыт).
- Не парсит motor-talk (GraphQL) и reddit (OAuth) — им нужны отдельные клиенты
  (см. docs/sources-scan.md), они не HTML-краул.
